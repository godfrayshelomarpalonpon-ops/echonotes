# EchoNotes: A Social Literary Engine
## Capstone Project Technical Documentation

**System Title:** EchoNotes  
**Domain:** Social Networking & Literary Creativity  
**Core Technologies:** Python, Django, SQLite, Google Gemini AI (Vertex AI/GenAI)

---

## 1. Executive Summary
EchoNotes is a comprehensive social platform designed specifically for the Filipino literary community. It combines traditional social networking features (profiles, following, direct messaging) with advanced gamification (badges, streaks) and cutting-edge AI integration. The platform uses AI to generate daily writing prompts, moderate content for toxicity, provide automated creative critique, and power "AI Personas" that interact with user content to foster engagement.

---

## 2. System Architecture
The system is built on a **Model-View-Template (MVT)** architecture:
- **Core Framework:** Django 4.x
- **Database:** SQLite (Development) / PostgreSQL compatible
- **AI Engine:** Google Gemini Flash (via `google-generativeai`)
- **Frontend:** Vanilla CSS / HTML5 / FontAwesome

---

## 3. Database Schema (Models)
The database is composed of 22 interconnected models managed via Django's ORM:

### 3.1 Content & Organization
- **Category (Circles):** Defines sub-communities (e.g., "Poetry", "Short Stories"). Features custom branding (icons/colors) and membership logic.
- **Post (Echoes):** The primary content unit. Includes fields for AI-generated summaries, toxicity scores, and mood tags.
- **Comment:** Nested interaction for posts with automated AI moderation.

### 3.2 Engagement Features
- **DailyPrompt:** A centralized repository for AI-generated writing challenges indexed by date.
- **WordOfTheDay:** A daily vocabulary challenge with definitions and community usage entries.
- **Contest:** An administrative module for hosting writing competitions with voting and deadline logic.

### 3.3 Gamification & Progression
- **WritingStreak:** Tracks consecutive days of user activity to reward consistency.
- **Badge / UserBadge:** A reward system that automatically grants icons (e.g., "Master Storyteller") based on user milestones.
- **LeaderboardEntry:** Cached rankings for "Most Liked," "Most Active," and "Contest Winners."

### 3.4 Social & Privacy
- **DirectMessage:** Private peer-to-peer communication with read-tracking.
- **Friend / FriendRequest:** Mutual-acceptance relationship logic.
- **Notification:** Real-time alerts for likes, comments, and social requests.

## 4. Functional Overview (Views)

### 4.1 Authentication & Profile Management
- `register()` / `login_view()` / `logout_view()`: Standard secure user lifecycle using Django's Auth system.
- `profile_detail()`: Aggregates user stats, achievements (badges), and their published "Echoes."

### 4.2 The Dashboard (Central Intelligence)
- `dashboard()`: The heart of the application. It performs complex queryset aggregation to provide:
  - **Activity Feed:** Recent interactions from people the user follows.
  - **Circle Feed:** Content from the user's joined communities.
  - **Global Discovery:** New content from across the platform.
  - **Social Sidebar:** Pending friend requests and community statistics.

### 4.3 AI Integration Logic
- `ai_broadcast()`: Fetches the latest system-wide message generated by the AI Persona Engine.
- `generate_prompt_ai()`: Uses Gemini to create evocative, context-aware writing prompts based on a theme.
- `ai_critique()`: A specialized endpoint that allows users to submit drafts for instantaneous literary feedback from an AI mentor.
- **Signals (`post_save`):** When a user publishes a post, a background signal triggers:
  1. **AIService.generate_summary:** Creates a one-sentence summary.
  2. **AIService.moderate_content:** Checks for toxicity.
  3. **AIPersonaEngine.interact_with_post:** Randomly selects 1-2 AI "personas" (like "The Encouraging Peer") to leave a comment or a like.

### 4.4 Social Interaction
- `follow_user()`: Implements a one-way follow relationship.
- `send_friend_request()` / `accept_friend_request()`: Implements a two-way mutual relationship.
- `conversation()`: A real-time enabled DM interface using AJAX polling for seamless communication.

## 5. Algorithmic Highlights

### 5.1 Writing Streak Logic
Located in `models.py (WritingStreak)`, the `update_streak()` method calculates the delta between the current date and `last_post_date`.
- If the delta is 1 day, the streak increments.
- If the delta > 1 day, the streak resets to 1.
- This maintains a high "retention" factor for the platform.

### 5.2 AI Persona Engine
Located in `ai_service.py`, this engine defines specific "personalities" with unique comment templates and like probabilities. It uses string interpolation and random sampling to make interactions feel human-like and dynamic.

### 5.3 Word of the Day Parsing
Located in `ai_monitor.py`, the system robustly parses JSON responses from the Gemini API, using regex-based clean-up to handle markdown formatting and ensure the database remains populated even if the API returns non-standard text.
